# Contributing to OpenPGxLab

Thank you for your interest in contributing to **OpenPGxLab**.

This project is being built as an open learning and development hub for pharmacogenomics (PGx), covering education, sequencing technologies, bioinformatics, HLA, annotation resources, and clinical interpretation.

## Ways to Contribute

You can contribute by helping with:

- improving explanations
- correcting errors
- suggesting better structure
- adding examples and diagrams
- proposing hands-on exercises
- improving technical accuracy
- expanding content related to sequencing, bioinformatics, HLA, annotation, and decision support

## General Principles

Please try to keep contributions:

- scientifically accurate
- technically clear
- well-structured
- practical
- accessible to both learners and technical users

## Content Style

When contributing written material:

- use clear section titles
- keep explanations structured
- define technical terms when needed
- avoid unnecessary jargon when a simpler explanation is possible
- prefer practical examples over abstract statements

## Markdown Structure

For consistency, please:

- use `#` for the page title
- use `##` for main sections
- use `###` for subsections
- use bullet points for short lists
- keep paragraphs readable and not too dense

## Modules and Exercises

If you want to add a new learning module or exercise:

- place modules under `docs/modules/`
- place exercises under `docs/exercises/`
- use descriptive file names
- keep the structure consistent with existing material

Examples:

- `docs/modules/module-02-core-genes.md`
- `docs/exercises/exercise-01-basic-pgx-cases.md`

## Suggestions and Corrections

If you notice an issue, you can contribute by:

- opening an issue
- suggesting a correction
- submitting a pull request
- proposing a new section or example

## Scope of the Project

OpenPGxLab may include material related to:

- PGx fundamentals
- PGx genes and mechanisms
- HLA and immune-mediated PGx
- sequencing technologies
- short-read and long-read analysis
- star alleles and haplotypes
- annotation databases
- clinical reporting
- population-specific PGx
- bioinformatics workflows and decision support

## Respect the Project Direction

OpenPGxLab is intended to remain:

- educational
- practical
- technically rigorous
- community-friendly
- open to interdisciplinary collaboration

## Thank You

Thank you for helping improve OpenPGxLab.
